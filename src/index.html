Git & Github | A version control system

Preface

OK, so there are a lot, and I mean A LOT, of tutorials on how to get started with Git and Github.com. And, for someone who is new to this whole 'front-end developer' stuff, the tuts can be confusing. So my next pet project is to write one with plain English. For now though, I hope this list will be enough to motivate you to, 'Git your shit together.'

Why use it?

  • History

    • Git/Github keeps a central copy or a 'master' of your project for you. You can work from a branch or "clone" of your project with out over writing your central project every time.

  • Collaboration

    • As a beginner, this one is important for me. My friends, coworkers, or anyone else that wants to help, can make a copy of my project. They can review my project and recommend changes using without modifying the central project. Then, I can incorporate their suggestions into my project with ease. Just by merging the branch and master.

  • Speed and Agility

    • After the initial learning curve, Git is fast, and it's quite powerful. The creator, Linus Torvalds (creator of Linux), designed Git with transparency in mind. It's easy for simple control but powerful enough to handle integrated commands. This flexibility comes in handy on large, custom projects.
